# Пункт 4.1: middleware для интеграции в execution pipeline Hot Chocolate

## Как реализовано
Добавлен `ExecutionProfilerMiddleware`, который:
1. решает, включен ли профайлер для текущего запроса,
2. создает `ExecutionProfileCollector`,
3. после выполнения запроса добавляет `extensions.profiling`,
4. публикует метрики/агрегаты.

## Основные файлы
- `HotChocolate/Core/src/Types/Execution/Pipeline/ExecutionProfilerMiddleware.cs`
- `HotChocolate/Core/src/Types/Execution/DependencyInjection/RequestExecutorBuilderExtensions.Profiling.cs`

## Ключевые цитаты
```csharp
context.SetExecutionProfilerExecutionState(enabled: true);
var profileCollector = new ExecutionProfileCollector(options);
context.Features.Set(profileCollector);
```

```csharp
var profilingExtension = profileCollector.CreateResultExtension(options, aggregates);
operationResult.Extensions =
    operationResult.Extensions.SetItem(
        ExecutionProfileCollector.ExtensionKey,
        profilingExtension);
```

```csharp
.UseRequest(
    ExecutionProfilerMiddleware.Create(),
    after: WellKnownRequestMiddleware.InstrumentationMiddleware,
    allowMultiple: false);
```
