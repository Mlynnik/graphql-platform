# Пункт 4.4: минимальный overhead в production (<5%)

## Как реализовано
1. В middleware есть fast-path для выключенного профайлера и для неподходящих операций (ранний `return` без профилирования).
2. Добавлен benchmark A/B (`profiler off` vs `profiler on`) для измерения накладных расходов.

## Основные файлы
- `HotChocolate/Core/src/Types/Execution/Pipeline/ExecutionProfilerMiddleware.cs`

- `HotChocolate/Core/benchmarks/Execution.Profiling.Benchmarks/Program.cs`

## Ключевые цитаты
```csharp
if (!isEnabled)
{
    context.SetExecutionProfilerExecutionState(enabled: false);
    await _next(context).ConfigureAwait(false);
    return;
}
```

```csharp
if (TryGetOperationIdentity(context, out var knownOperationType, out var knownOperationName)
    && !options.ShouldProfileOperation(knownOperationType, knownOperationName))
{
    context.SetExecutionProfilerExecutionState(enabled: false);
    await _next(context).ConfigureAwait(false);
    return;
}
```
