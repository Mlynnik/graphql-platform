# Пункт 3.5: группировка статистики по операциям (query/mutation/subscription)

## Как реализовано
Тип и имя операции извлекаются в middleware, сохраняются в `ExecutionProfilerRequestSample`, а агрегатор строит группы:
- `byOperationType`
- `byOperation` (тип + имя)

## Основные файлы
- `HotChocolate/Core/src/Types/Execution/Pipeline/ExecutionProfilerMiddleware.cs`
- `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerRequestSample.cs`
- `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerSlidingWindowAggregator.cs`
- `HotChocolate/Core/test/Execution.Tests/DependencyInjection/RequestExecutorBuilderExtensions_ExecutionProfiler.Tests.cs`

## Ключевые цитаты
```csharp
var operationType = GetOperationType(context);
var operationName = GetOperationName(context);
var requestSample = profileCollector.CreateRequestSample(operationType, operationName);
```

```csharp
internal sealed record ExecutionProfilerRequestSample(
    DateTimeOffset CapturedAtUtc,
    string OperationType,
    string? OperationName,
    ...
);
```

```csharp
["byOperationType"] = SerializeByOperationType(byOperationType),
["byOperation"] = SerializeByOperation(byOperation),
```
