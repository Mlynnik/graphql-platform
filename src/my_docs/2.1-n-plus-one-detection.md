# Пункт 2.1: автоматическое обнаружение потенциальных N+1 при отсутствии batching через DataLoader

## Что требовалось
Автоматически находить потенциальные N+1-сценарии, когда одно и то же поле внутри списка выполняется много раз без признаков batching/caching через DataLoader.

## Как реализовано

Реализация собрана в профайлере исполнения и состоит из трех частей:

1. Сбор сигнала о DataLoader batching/cache на уровне текущего поля.
2. Нормализация путей полей со списками (`users[0].x`, `users[1].x` -> `users[].x`) и подсчет повторов.
3. Формирование `nPlusOne.issues` только для паттернов без DataLoader batching/cache.

## В каких файлах сделано

1. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerDiagnosticEventListener.cs`
- Привязка активного `Path` к текущему resolver scope.

2. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerScopeContext.cs`
- `AsyncLocal<Path?>` для передачи текущего пути в DataLoader listener.

3. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerDataLoaderDiagnosticEventListener.cs`
- Запись фактов cache hit и batch execution в `ExecutionProfileCollector`.

4. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfileCollector.cs`
- Алгоритм детекции N+1 (`AnalyzeNPlusOneIssues`), фильтрация ложных срабатываний при наличии DataLoader batching/cache, формирование payload с предупреждением и рекомендацией.

5. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerOptions.cs`
- Порог срабатывания `NPlusOneListPatternThreshold`.

6. `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerServiceCollectionExtensions.cs`
- Поддержка конфигурации порога через `IConfiguration`.

7. `HotChocolate/Core/test/Execution.Tests/DependencyInjection/RequestExecutorBuilderExtensions_ExecutionProfiler.Tests.cs`
- Интеграционные тесты детекции N+1 и detail-level поведения.

## Чем подтверждено тестами

Файл: `RequestExecutorBuilderExtensions_ExecutionProfiler.Tests.cs`

1. `ExecuteAsync_Should_AddNPlusOneIssue_When_RepeatedIndexedPathHasNoDataLoaderBatching` (строки 472-492)
- Проверяет, что при повторяющемся индексированном пути появляется `nPlusOne.issueCount = 1` и issue для `users[].profileName`.

2. `ExecuteAsync_Should_NotAddNPlusOneSection_When_DetailLevelIsSlowFields` (строки 495-512)
- Проверяет, что секция `nPlusOne` не добавляется в режиме `SlowFields`.

3. `ExecuteAsync_Should_IncludeOnlyNPlusOnePayload_When_DetailLevelIsNPlusOneOnly` (строки 515-537)
- Проверяет режим, когда возвращается только N+1 payload без полной field-статистики.
