# Пункт 5.1: пороговые значения для алертов

## Как реализовано
В опциях профайлера заведены пороги:
- `SlowFieldThreshold`
- `SlowRequestThreshold`
- `SlowRequestFieldLimit`
- `NPlusOneListPatternThreshold`

Эти пороги применяются при формировании `profiling` и slow-request warning логов.

## Основные файлы
- `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerOptions.cs`
- `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfileCollector.cs`
- `HotChocolate/Core/src/Types/Execution/Pipeline/ExecutionProfilerMiddleware.cs`
- `HotChocolate/Core/src/Types/Execution/Profiling/ExecutionProfilerServiceCollectionExtensions.cs`

## Ключевые цитаты
```csharp
public int NPlusOneListPatternThreshold { get; set; } = 3;
public TimeSpan SlowFieldThreshold { get; set; } = TimeSpan.Zero;
public TimeSpan SlowRequestThreshold { get; set; } = TimeSpan.FromMilliseconds(500);
public int SlowRequestFieldLimit { get; set; } = 5;
```

```csharp
var thresholdNanoseconds = options.SlowRequestThreshold.Ticks * 100L;
if (requestSample.RequestDurationNanoseconds < thresholdNanoseconds)
{
    return;
}
```

```csharp
if (int.TryParse(section["NPlusOneListPatternThreshold"], out var threshold)
    && threshold > 0)
{
    options.NPlusOneListPatternThreshold = threshold;
}
```
